스타트업이든 대규모 엔터프라이즈 개발팀에 속해 있든 상관없이 시간, 비용, 리소스를 절약해야 합니다. 엘릭서와 라이브뷰 네이티브가 이를 해결해 드립니다. 엘릭서가 더 나은 디지털 제품을 더 빠르고 효율적으로 출시하는 데 도움이 되는 모든 방법을 무료 전자책 '엘릭서의 비즈니스 가치'를 통해 알아보세요.

Elixir에서 멋진 새 웹 앱에 대한 마지막 몇 가지 LOC를 작성했는데, 기술 제품 관리자가 방금 프로덕션에 배포한 백엔드에 연결되는 네이티브 모바일 앱을 개발하는 데 필요한 스토리 포인트가 몇 개인지 묻는 깜짝 핑을 Jira에서 받게 됩니다. 또한 프레임워크로 TODO 앱을 빌드하는 방법에 대한 YouTube 동영상을 보고 깔끔해 보인다는 이유로 이 모바일 앱을 React Native로 빌드하는 것이 좋겠다고 제안합니다.

좌절감은 잠시 접어두고 다음과 같은 질문으로 답합니다: React 또는 다른 프레임워크로 구축된 네이티브 모바일 앱에 얼마를 지출할 의향이 있나요?

PM 동료에게 물어보고 싶지만 스스로에게 묻고 싶은 질문은 다음과 같습니다: 라이브뷰 네이티브가 옵션인데 왜 같은 작업을 두 번 해야 하는가? 이를 무시한다면 얼마나 많은 것을 포기해야 할까요?

라이브뷰 네이티브의 불로장생의 필요성

2019년에 피닉스 프레임워크 개발자들은 실시간 웹 애플리케이션을 구축하기 위한 서버 중심 모델인 피닉스 라이브뷰를 출시했습니다. 라이브뷰는 처음에는 일반 HTTP 요청을 통해 페이지를 정적으로 렌더링한 다음, 서버에서 변경 사항이 발생하면 데이터와 상태를 실시간으로 동기화하는 클라이언트와 서버 간의 영구 WebSocket 연결로 전환합니다.

여러 도구와 프레임워크가 필요했던 개발 환경을 효과적으로 통합하는 라이브뷰의 장점 때문에 Elixir 에코시스템에 이미 참여하고 있는 개발자와 그 외의 개발자가 모두 라이브뷰로 전환했습니다. 사용자 측면에서도 라이브뷰는 페이로드와 지연 시간을 제한하기 위해 전체 웹페이지가 아닌 변경된 데이터의 델타만 전송하므로 더욱 빠르고 원활한 경험을 제공합니다.

독야드 팀은 2022년에 피닉스 라이브뷰의 후속 버전인 라이브뷰 네이티브를 출시하여 하나의 피닉스 라이브뷰가 여러 팀을 조율하면서 파편화된 프레임워크에서 동일한 비즈니스 로직을 여러 번 작성할 필요 없이 iOS 및 Android 기기용 SwiftUI와 Jetpack으로 작성된 클라이언트를 포함한 여러 클라이언트와 상호 작용할 수 있도록 했습니다.

라이브뷰가 엘릭서를 사용하여 개발할 수 있는 사람들의 폭을 넓혔다면, 라이브뷰 네이티브는 네이티브 모바일 앱의 미래에 큰 영향을 미쳤습니다.

Elixir와 네이티브 앱의 믹스 앤 매치 개발 비용 비교

라이브뷰와 라이브뷰 네이티브가 나오기 전에는 기존 엘릭서 백엔드에 연결하기 위해 완전히 새로운 소프트웨어 개발 라이프사이클(SDLC)인 두 번째 프레임워크와 언어를 선택해야 했습니다. 새로운 SDLC를 사용하면 새로운 기능 개발에 소요되는 시간 W + 엔지니어링 및 운영에 소요되는 시간 X + 테스트에 소요되는 시간 Y + 새롭고 개별적인 버그를 수정하는 데 소요되는 시간 Z가 추가됩니다.

또한 이 모든 작업을 관리하기 위해 완전히 별도의 팀이 필요하며, 이러한 인력을 채용하는 것만으로도 모바일 앱 구축에 아무리 넉넉한 예산을 책정해도 소진될 수 있습니다.

네이티브 앱 개발에 다른 프레임워크를 사용할 경우의 비용을 설명하기 위해 유명 소프트웨어 개발 에이전시 세 곳에 네이티브 모바일 애플리케이션 구축에 대한 견적을 요청했습니다. 작업 범위에는 사용자 대면 모바일 애플리케이션과 실시간 채팅 환경을 만들고 데이터를 유지하는 데 필요한 백엔드가 모두 포함되었습니다.

제품 요구 사항 문서(PRD)에 따르면 사용자는 iOS 또는 웹 클라이언트와 여러 기기에서 동시에 고유한 6자리 코드를 통해 로그인해야 합니다. 인증이 완료되면 사용자는 회의 일정별로 사용 가능한 채팅방을 확인하고, 채팅하고, 지속적인 채팅 기록을 확인하고, 링크나 미디어를 공유할 수 있어야 합니다.

대행사 #1

이 기관은 프런트엔드에 React를 사용하고 백엔드로 Elixir를 사용하는 옵션과 Swift와 Elixir를 사용하는 옵션 두 가지를 제안했습니다.

React+Elixir의 경우 리드 개발자의 경우 680시간, 백엔드 엔지니어의 경우 640시간이 소요될 것으로 예상했으며, 완료까지 17주 이상이 소요될 것으로 예상했습니다. Swift로 개발하면 리드 개발자는 880시간, 백엔드 엔지니어는 640시간으로 프로젝트가 22주로 늘어났습니다.

에이전시 #2

두 번째 대행사는 프론트엔드에는 React를, 백엔드에는 Node.js를 사용하는 다른 접근 방식을 제안했으며, 3명의 선임 iOS/백엔드/React 개발자가 각각 480시간과 12주 동안 PRD의 요구 사항을 충족하는 데 소요되었습니다.

에이전시 #3

세 번째 에이전시에서는 동일한 React+Node.js 프레임워크를 사용하여 소프트웨어 엔지니어 3명에게 400시간, QA 테스터 1명에게 150시간으로 10주를 요청했습니다.

라이브뷰 네이티브

웹 프론트엔드, 모바일 프론트엔드, 백엔드에 단일 Elixir 코드베이스를 사용하는 LiveView Native를 사용하여 동일한 애플리케이션을 개발하는 데 필요한 견적은 리드 개발자가 320시간, QA 테스터가 80시간으로 8주밖에 걸리지 않습니다.

비용 계산

대행사가 예상하는 가장 빠른 출시 기간은 17주, 1,300시간 이상이며, 이는 필연적으로 조직에 상당한 비용을 초래할 것입니다. 이는 PRD에 따라 애플리케이션을 구축하는 데만 소요되는 시간이며, 버그 수정이나 지속적인 운영 지원은 포함되지 않습니다.

라이브뷰 네이티브는 다른 네이티브 앱 프레임워크를 사용할 때보다 개발 시간 및 비용을 40% 이상 절약할 수 있습니다.

라이브뷰 네이티브의 비용 절감 효과는 단순히 타임라인이 단축되고 청구 가능한 시간이 줄어드는 것뿐만이 아닙니다. 여러 언어와 프레임워크로 분산되어 별도로 관리해야 하는 사일로화된 SDLC를 생성하는 대신, 단일 코드 기반과 팀을 사용하여 이미 강력한 Elixir 기반에서 네이티브 앱 기능을 구축할 수 있습니다.

비용 이상의 효과: 모바일 앱용 Elixir의 이점

모바일 앱을 출시하기 위한 백지 수표가 있는데 기술 제품 관리자가 처음으로 프로덕션에 배포할 구체적인 일정을 요구하지 않는다고 가정해 보겠습니다. 이 시나리오에서는 에이전시 제안 중 하나를 받아들여 작업을 완전히 아웃소싱할 수 있습니다.

이 경우에도 Elixir는 개발 기간과 첫 라이브 출시 이후 훨씬 더 큰 이점을 제공합니다.

  • 라이브뷰 네이티브 애플리케이션이 더 빠르고 가벼워집니다. 엘릭서의 불변성 덕분에 라이브뷰 네이티브는 지연 시간이 짧은 웹 소켓 연결을 사용하여 서버 측 변경 시 가능한 한 최소한의 데이터를 전송하여 사용자가 원하는 반응적이고 만족스러운 경험을 제공하며, 특히 실시간 경험에서 더욱 그렇습니다.

  • 보다 일관된 사용자 경험을 제공합니다. 라이브뷰 네이티브 앱은 플랫폼에 구애받지 않는 엘릭서 코드를 네이티브 사용자 인터페이스로 변환하므로 프로덕션급 웹과 모바일 앱이 마치 하나의 앱인 것처럼 느껴집니다.

  • 릴리스 주기가 단축됩니다. Elixir 코드베이스의 단순성과 통합성 덕분에 버그 수정과 보안 개선을 더 빠르게, 더 큰 최종 사용자에게 영향을 미치면서 푸시할 수 있으므로 MVP를 구축하거나 기술 부채를 줄이는 데 집중할 수 있습니다.

  • 앱에 더 많은 실시간 기능을 추가할 수 있습니다. 분산된 워크로드와 이벤트를 처리하기 위해 메시지 대기열이나 외부 데이터베이스를 추가할 필요 없이 Elixir와 LiveView Native의 서버 중심 모델을 사용하면 시스템 내 어디에서 이벤트가 발생하든 최대한 빨리 사용자에게 푸시할 수 있습니다.

  • 자신 있게 구축하세요. Elixir와 Erlang VM을 사용하면 실수로 애플리케이션에 발생한 오류나 충돌이 백엔드를 다운시키고 최종 사용자 경험을 저해하지 않도록 구분하여 처리할 수 있습니다.

  • 더욱 탄력적이고 확장 가능한 시스템을 구축할 수 있습니다. Elixir의 기본 제공 동시성, 오류 보고, 테스트, 성능 프로파일링 기능은 수직적 또는 수평적 규모에 맞게 최적화하는 데 도움이 됩니다.

Elixir 에코시스템을 사용하면 전략적인 작업에 더 많은 시간을 할애할 수 있습니다. 조직의 기술에 지속적인 영향을 미칠 수 있는 설계에 더 많은 시간을 할애하는 것이야말로 가장 큰 장애물을 제거할 수 있는 방법일 수 있습니다.

결론

라이브뷰 네이티브에 대한 최신 소식을 확인하려면 Elixir Lang 슬랙의 #liveview-native 채널에 가입하거나 GitHub에서 진행 중인 개발에 참여할 수 있습니다.

실험을 통해 Elixir를 통해 얻을 수 있는 비용 절감과 효율성 향상에 대해 충분히 확신하지 못하셨다면, 수백만 달러 규모의 비즈니스에서 개발자의 만족도와 Elixir의 동시성 및 실시간 인터랙티브 기능이 어떻게 큰 성과를 거두고 있는지에 대한 실제 데이터를 담은 전자책 'Elixir의 비즈니스 가치'를 계속 읽어보시기 바랍니다.
