소개

이 글에서는 HTMX의 정의와 그 기능에 대해 살펴봅니다.

HTMX는 하이퍼텍스트(마크업)의 쉽고 강력한 기능으로 최첨단 사용자 인터페이스를 만들 수 있는 종속성이 없는 작은(14k 분, 압축 파일 크기) 자바스크립트 라이브러리입니다. 속성을 사용하여 HTML에서 직접 AJAX, CSS 전환, 웹소켓 및 서버 전송 이벤트에 액세스할 수 있는 기능을 제공합니다. 이는 마크업만으로 (자바스크립트가 제공하는) 상호작용을 구현할 수 있게 해주어 개발자의 판도를 바꾸어 놓았습니다.

이 문서에서 따라야 할 단계는 다음과 같습니다:

  • HTMX를 사용한 유효성 검사
    • HTMX를 사용한 애니메이션
    • HTMX의 확장 기능
    • HTMX의 로깅 및 이벤트
      • 이벤트
    • 로깅

HTMX 설치

세 가지 방법으로 HTMX를 설치할 수 있습니다:

CDN을 통해: HTML 파일 헤드에 CDN을 포함시켜 웹사이트/애플리케이션에 HTMX를 로드할 수 있습니다.

<script
  src="https://unpkg.com/htmx.org@1.9.6"
  integrity="sha384-FhXw7b6AlE/jyjlZH5iHa/tTe9EpJ1Y55RjcgPbjeWMskSxZt1v9qkxLJWNJaGni"
  crossorigin="anonymous"
></script>

N/B: 이 방법은 가능하지만 프로덕션 환경에서는 CDN을 사용하지 않는 것이 좋습니다.

웹사이트/애플리케이션에 HTMX 소스 파일의 사본을 다운로드할 수 있습니다.

htmx.min.js 에서 다운로드하여 프로젝트의 적절한 디렉터리에 넣은 다음 필요에 따라 스크립트> 태그와 함께 포함시킵니다:

<script src="/path/to/htmx.min.js"></script>

npm을 통해 설치: 아래 명령어를 사용하여 npm을 통해 HTMX를 설치할 수 있습니다.

설치 후 적절한 툴을 사용하여 node_modules/htmx.org/dist/htmx.js 또는 node_modules/htmx.org/dist/htmx.min.js) 을 활용해야 합니다. 예를 들어, 특정 확장 프로그램 및 프로젝트별 코드와 함께 HTMX를 패키징할 수 있습니다.

웹팩에서 HTMX 설정하기: 웹팩을 사용하여 자바스크립트를 관리하는 경우 먼저 패키지 관리자(npm, yarn, pnpm)를 통해 htmx를 설치한 다음 가져오기를 추가합니다. index.js

전역 htmx 변수를 사용하려면 다음과 같이 윈도우 스코프에 삽입하면 됩니다:

  • 고유한 JS 파일을 만듭니다.
  • 이 파일을 index.js 파일로 가져옵니다.
import "path/to/my_custom.js";
  • 그런 다음 파일에 아래 코드를 추가합니다.
window.htmx = require("htmx.org");
  • 마지막으로 애플리케이션 번들을 다시 빌드합니다(변경 사항을 반영하기 위해).

HTMX를 사용한 Ajax 요청

HTMX는 HTML에서 직접 Ajax 요청을 수행할 수 있는 사용자 정의 속성을 제공합니다:

속성설명
hx-post지정된 URL에 포스트 요청을 실행합니다.
hx-get지정된 URL로 get 요청을 실행합니다.
hx-put지정된 URL에 대한 풋 요청을 실행합니다.
hx-patch는 지정된 URL로 패치 요청을 실행합니다.
hx-delete는 지정된 URL로 삭제 요청을 실행합니다.

사용사례

<div hx-post="/messages">Put To Messages</div>

위의 코드에서 hx-post 속성은 해당 URL에 대한 포스트 API 호출을 트리거합니다. "/messages"

트리거요청

HTMX에서는 요소의 자연스러운 이벤트가 AJAX 요청을 시작합니다. 예를 들어, onchange 이벤트는 input, select, textarea 을 트리거하고, onsubmit 이벤트는 form 을 트리거하며, onclick 이벤트는 그 외 모든 것을 트리거합니다.

그러나 HTMX는 요청을 시작하는 이벤트를 변경하려는 경우를 위해 고유한 hx-trigger 속성을 제공합니다:

<div hx-post="http://localhost:3000/submit" hx-trigger="mouseenter">
  Submit
</div>

위 코드에서는 사용자의 마우스가 div 위에 있는 경우에만 지정된 URL로 GET 요청이 이루어집니다.

트리거 수정자

트리거의 동작은 몇 가지 다른 수정을 통해 변경할 수도 있습니다. 예를 들어 요청이 한 번만 발생하도록 하려면 트리거에 once 수정자를 사용합니다:

<div hx-post="/mouse_entered" hx-trigger="mouseenter once">
  Submit
</div>

사용할 수 있는 다른 트리거 수정자는 다음과 같습니다.변경됨 - 요소의 값이 변경된 경우에만 요청을 보냅니다.

delay:<time interval> - 요청을 하기 전 지연 기간을 지정합니다. 이벤트가 다시 발생하면 카운트다운이 초기화됩니다.

스로틀:<interval time> - 요청을 하기 전 지연 시간을 지정합니다. 지연과 달리 시간 제한에 도달하기 전에 새 이벤트가 발생하면 이벤트가 삭제되고 해당 기간이 끝나면 요청이 트리거됩니다.

출처:<CSS Selector>- 다른 요소의 이벤트를 수신합니다.

HTMX가 지원하는 더 많은 수정자를 보려면 여기를 방문하세요.

트리거필터

이벤트 이름 뒤에 대괄호로 자바스크립트 표현식을 둘러싸서 적용합니다. 표현식이 참으로 평가되면 이벤트가 트리거되고, 그렇지 않으면 트리거되지 않습니다.

<div hx-get="/clicked" hx-trigger="click[ctrlKey]">
  Click the force
</div>

htmx에는 hx-trigger와 함께 사용할 수 있는 몇 가지 특수 이벤트가 포함되어 있습니다:

  • load - 요소가 처음 로드될 때 한 번 발생합니다.
  • revealed - 요소가 뷰포트에 처음 스크롤될 때 발생합니다.
  • intersect - 요소가 뷰포트와 처음 교차할 때 발생합니다. 여기에는 두 가지 추가 옵션이 있습니다:
    • root:<selector> - 교차점에 대한 루트 요소 CSS 선택기
    • 임계값<float> - 0.0에서 1.0 사이의 부동 소수점 숫자로, 이벤트가 트리거될 교차점의 양을 나타냅니다.

고급 사용 사례가 있는 경우 사용자 정의 이벤트를 활용하여 요청을 트리거할 수도 있습니다.

폴링

를 사용하면 every 구문과 시간 프레임(초 단위)을 hx-trigger 속성과 함께 사용하여 요소가 특정 URL을 폴링하도록 만들 수 있습니다:

<div hx-get="/get-star-wars-data" hx-trigger="every 2s"></div>

로드 폴링

"로드 폴링"은 HTMLX에서 폴링을 수행하기 위한 추가 방법입니다. 여기에는 요소가 로드 트리거와 지연을 지정한 다음 스스로 응답으로 대체하는 것이 포함됩니다:

<div hx-get="/get-star-wars-data" hx-trigger="load delay:1s" hx-swap="starWarsDataHTML"></div>

요청 표시기

브라우저는 AJAX 요청이 이루어질 때 피드백을 제공하지 않기 때문에 사용자에게 어떤 일이 일어나고 있는지 알려주는 것이 유용할 때가 많습니다. HTMX에서는 htmx-indicator 클래스를 사용하여 이를 수행할 수 있습니다.

htmx-indicator 클래스가 있는 요소의 불투명도는 기본적으로 0으로 설정되며, 이는 해당 요소가 DOM에 존재하더라도 보이지 않음을 의미합니다.

HTMX가 요청을 제출할 때 htmx-request 클래스가 요소에 추가됩니다. htmx-indicator 클래스가 있는 자식 요소가 htmx-request 클래스에 노출되면 불투명도 1로 전환되어 표시기를 표시합니다.

<button hx-get="/click">
    Click Me!
    <img class="htmx-indicator" src="/spinner.gif">
</button>

원하는 경우 hx-indicator 속성과 CSS 선택기를 사용하여 htmx-request 클래스를 대체 요소에 추가할 수 있습니다:

<div>
  <button hx-get="/click" hx-indicator="#indicator">
    Click the force!
  </button>
  <img id="indicator" class="htmx-indicator" src="/spinner.gif" />
</div>

대상

CSS 선택기(div, class 등)를 허용하는 hx-target 속성을 사용하여 요청을 시작한 요소가 아닌 다른 요소에 응답을 로드할 수 있습니다.

<input type="text" name="star-wars-input"
    hx-get="/star-wars-characters"
    hx-trigger="keyup delay:500ms changed"
    hx-target="#search-results"
    placeholder="Search..."
>
<div id="search-results"></div>

위의 예에서 hx-target 속성은 #search-result 아이디를 대상으로 하며 요청이 성공하면 해당 아이디가 포함된 div에 요청 값을 채웁니다.

스와핑

HTMX에서 DOM으로 반환된 HTML 콘텐츠를 바꿀 수 있습니다. 대상 요소의 innerHTML 가 자동으로 콘텐츠로 대체됩니다. 이 값은 hx-swap 속성에 다음 값 중 하나를 적용하여 변경할 수 있습니다:

이름설명
innerHTML이면 기본적으로 콘텐츠가 대상 요소 안에 삽입됩니다.
outerHTML이면 반환되는 콘텐츠가 대상 요소를 완전히 대체합니다.
afterbegin콘텐츠가 대상의 첫 번째 자식 앞에 추가됩니다.
beforebegin은 대상의 부모 요소의 콘텐츠를 대상 앞에 추가합니다.
beforeend는 대상의 마지막 자식 뒤에 콘텐츠를 추가합니다.
afterend는 대상 뒤에 있는 대상의 부모 요소에 콘텐츠를 추가합니다.
delete응답에 관계없이 대상 요소를 제거합니다.
none응답의 콘텐츠를 포함하지 않습니다.

HX-SWAP의 동작 예시

<div hx-get="/example" hx-swap="afterend">
  Get Some HTML & Append It
</div>

위의 코드에서 div 는 요청을 하고 반환된 콘텐츠를 div 뒤에 추가합니다.

동기화

두 요소에서 요청을 할 때 그 중 하나가 다른 요소보다 우선순위를 갖거나 다른 요소의 요청이 완료될 때까지 기다리려는 경우 HTMX에는 hx-sync 기능이 유용할 수 있습니다.

예를 들어, 양식 제출과 특정 입력에 대한 유효성 검사 요청 간에 경쟁 상황이 발생하는 시나리오를 그려 보겠습니다:

<form hx-post="/submit-character">
    <input id="title" name="title" type="text"
        hx-post="/validate-character"
        hx-trigger="change"
    >
    <button type="submit">Submit</button>
</form>

입력이 완료되고 양식이 즉시 제출되면 hx-sync를 사용하지 않고 /validate-character 및 /submit-character 으로 두 개의 요청이 동시에 전송됩니다.

입력에 hx-sync="closest form:abort "를 적용하면 양식에서 요청을 감시하고 입력 요청이 처리되는 동안 양식 요청이 발견되거나 시작되면 입력 요청이 중단됩니다. 이렇게 하면 동기화 제약 조건이 자동으로 해결됩니다.

<form hx-post="/submit-character">
    <input id="title" name="title" type="text"
        hx-post="/validate-character"
        hx-trigger="change"
        hx-sync="closest form:abort"
    >
    <button type="submit">Submit</button>
</form>

또한 htmx는 요청을 취소하는 프로그래밍 방식을 제공합니다: 기내 요청을 중지하려면 요소에 htmx :abort 이벤트를 htmx.trigger 함수로 보내면 됩니다:

<button id="request-button" hx-post="/submit-character">
    Submit Character
</button>
<button onclick="htmx.trigger('#request-button', 'htmx:abort')">
    Cancel Submission
</button>

CSS 전환

HTMX를 사용하면 자바스크립트 없이도 CSS 전환을 수행할 수 있습니다. 예시를 통해 이를 설명하겠습니다:

<div id="div1">The Empire</div>

그런 다음 아약스 요청을 하고 콘텐츠를 아래 콘텐츠로 바꿉니다:

<div id="div1" class="red">
  The Republic
</div>

위 코드에서 새 콘텐츠에 새 클래스 red 가 추가되었습니다. 이 시나리오에서는 이전 상태( red 클래스 없이)에서 현재 상태( red 클래스 포함)로 CSS 전환을 만들 수 있습니다:

.red {
  color: red;
  transition: all ease-in 1s;
}

HTMX가 내부에서 전환을 처리하는 방법: 새 콘텐츠가 페이지에 추가되기 전에 현재 콘텐츠의 id 속성과 일치하는 요소가 발견됩니다. 이는 새 콘텐츠가 서버에서 수신될 때 발생합니다. 교체가 이루어지기 전에 업데이트된 콘텐츠에서 일치하는 속성이 발견되면 이전 콘텐츠의 속성이 새 요소에 복제됩니다. 그 후에는 새 콘텐츠가 교체되지만 이전 속성 값은 그대로 유지됩니다. 그런 다음 새 속성 값은'정착' 시간(기본적으로 20ms) 후에 전환됩니다.

대역 외 스왑

응답 HTML의 hx-swap-oob 속성은 id 속성을 사용하여 응답의 콘텐츠를 DOM으로 직접 교체하는 데 사용할 수 있습니다:

<div id="message" hx-swap-oob="true">Dark Sidious</div>
Anakin Skywalker

위 코드에서 추가 콘텐츠는 일반적인 방식으로 타깃으로 스왑되지만 div#message 은 이 응답의 해당 DOM 요소로 바로 스왑됩니다.

파라미터

hx-params 속성을 사용하여 AJAX 호출과 함께 제출될 인수를 필터링할 수 있습니다.

이 속성에 사용할 수 있는 값은 다음과 같습니다:

  • "*"- 기본적으로 모든 매개 변수를 포함합니다.

  • none - 포함하지 않음

  • <param-list> 매개 변수를 포함하지 않습니다. 쉼표로 구분되어야 하는 매개변수 이름 목록

  • <param-list>쉼표로 구분해야 함 모든 매개변수 이름을 쉼표로 구분한 목록을 제공합니다.

<div hx-get="/get-starwars-characters" hx-params="*">
  Luke Skywalker
</div>

위 예제에서는 POST에 포함될 수 있는 모든 파라미터가 이 div에 포함되지만, GET의 경우와 마찬가지로 URL로 인코딩되어 URL에 포함됩니다.

요청 확인

HTMX는 간단한 자바스크립트 대화 상자를 사용하여 동작을 확인할 수 있는 hx-confirm 속성을 제공합니다:

<button hx-delete="/delete-character" hx-confirm="Are you sure you wish to delete Obiwan Kenobi">
  Delete Obiwan Kenobi
</button>

웹 소켓 및 SSE

HTMX는 현재 웹 소켓 및 서버 전송 이벤트를 실험적으로 지원하고 있습니다. 그러나 웹소켓 연결을 설정하려면 hx-ws 속성을 사용하면 됩니다.

<div hx-ws="connect:wss:/darth-chatroom">
    <div id="chat_room">
        ...
    </div>
    <form hx-ws="send:submit">
        <input name="chat_message">
    </form>
</div>

SSE(서버 전송 이벤트)를 사용하여 브라우저에 이벤트를 보내려면 부모 요소에 connect <url> 선언을 추가하고 URL과 함께 hx-sse 속성을 추가합니다.

그런 다음 hx-trigger="sse:<event_name> 속성을 사용하여 이 요소의 하위 요소이며 서버에서 보낸 이벤트에 의해 활성화되는 요소를 정의합니다. 문법

샘플은 다음과 같습니다.

<body hx-sse="connect:/darth-events">
  <div hx-trigger="sse:new_news" hx-get="/news"></div>
</body>

히스토리 지원

요소에 hx-push-url 속성을 사용하면 브라우저의 탐색 모음에 요청 URL을 푸시하고 페이지의 현재 상태를 히스토리에 추가할 수 있습니다:

<a hx-get="/star-wars-characters" hx-push-url="true">
  Blog
</a>

hx-push-url 작동 원리: /star-wars-characters 으로 요청하기 전에 HTMX는 현재 문서의 스냅샷을 찍어 사용자가 이 링크를 클릭하면 저장합니다. 그 후 새 위치가 기록 스택에 추가되고 스왑이 완료됩니다.

HTMX는 사용자가 뒤로 버튼을 누르면 스토리지에서 이전 콘텐츠를 검색하여 대상에 다시 스왑함으로써 이전 상태로 '되돌아가는' 시뮬레이션을 수행합니다. true 캐시에서 해당 위치를 찾을 수 없는 경우 HTMX는 헤더가 HX-History-Restore-call 로 설정된 지정된 URL로 아약스 호출을 전송하고 전체 페이지에 필요한 HTML을 다시 기대합니다. 또는 htmx.config.refreshOnHistoryMiss 구성 설정이 true로 설정된 경우 브라우저를 강제로 새로 고칩니다.

HTMX를 사용한 유효성 검사

HTMX는 HTML5 유효성 검사 API와 함께 작동하므로 유효성을 검사할 수 있는 입력이 유효하지 않으면 양식 요청이 이루어지지 않습니다. 이는 웹소켓 전송과 AJAX 쿼리 모두에 적용됩니다.

HTML5의 유효성 검사와 관련된 이벤트는 다음과 같습니다:

htmx:validation:validate - elements checkValidity() 메서드가 호출되기 전에 호출됩니다. 사용자 정의 유효성 검사 로직을 추가하는 데 사용할 수 있습니다.

htmx:validation:failed - checkValidity()가 거짓을 반환하여 잘못된 입력을 나타낼 때 호출됩니다.

htmx:validation:halted - 유효성 검사 오류로 인해 요청이 발행되지 않을 때 호출됩니다. 특정 오류는 event.detail.errors 객체에서 확인할 수 있습니다.

<form hx-post="/create-characters">
    <input _="on htmx:validation:validate
                if my.value != 'foo'
                    call me.setCustomValidity('Please enter the value foo')
                else
                    call me.setCustomValidity('')"
        name="example"
    >
</form>

위의 코드는 htmx:validation을 사용한 입력의 예시입니다. 코드에서 하이퍼스크립트를 사용하여 이벤트의 유효성을 검사하여 입력에 foo 값이 있는지 확인했습니다.

HTMX를 사용한 애니메이션

HTMX는 CSS와 HTML만 사용하여 웹 페이지에 원활한 애니메이션과 전환을 추가할 수 있도록 고안되었습니다. 이제 새로운 뷰 트랜지션 API를 활용하여 HTMX로 애니메이션을 만들 수 있습니다. 기본적인 예는 스왑 시 페이드 아웃 애니메이션입니다. 요청이 완료되면 제거될 요소를 페이드 아웃하려면 일부 CSS와 함께 htmx-swapping 클래스를 사용하고 애니메이션이 완료될 때까지 스왑 단계를 충분히 길게 연장하면 됩니다. 이 작업은 다음과 같이 수행할 수 있습니다:

<style>
.fade-me-out.htmx-swapping {
  opacity: 0;
  transition: opacity 1s ease-out;
}
</style>

<button class="fade-me-out"
        hx-delete="/fade_out_demo"
        hx-swap="outerHTML swap:1s">
        Fade Me Out
</button>

HTMX를 사용한 애니메이션의 더 많은 예는 여기에 나와 있습니다.

HTMX의 확장 기능

HTMX에는 HTMX 기반 애플리케이션 내에서 확장 기능을 만들고 배포하기 위한 확장 프레임워크가 포함되어 있습니다. hx-ext 속성을 사용하면 확장 라이브러리를 활용할 수 있습니다(자바스크립트에서 지정한 후).

확장 프로그램을 사용하려면 두 단계를 거쳐야 합니다:

  • 확장 프로그램 정의를 포함시키면 HTMX 확장 프로그램 레지스트리에 추가됩니다.
  • 확장자를 지정하는 데는 hx-ext 속성이 사용됩니다.

아래에 예가 나와 있습니다:

// Including the extension library
<script src="/path/to/ext/debug.js" defer></script>

// add the extension(debug) to the hx-ext attribute
<button hx-post="/example" hx-ext="debug">
    This Button Uses The Force
</button>

HTMX는 일반적인 개발자의 요구 사항을 충족하는 확장 프로그램 세트와 함께 제공됩니다. 각 릴리스에서 이러한 확장 기능은 htmx에 대해 테스트됩니다. HTMX에 포함된 확장 프로그램 목록을 보려면 여기를 방문하세요.

HTMX의 로깅 및 이벤트

이벤트

HTMX에는 로깅 시스템 역할도 하는 강력한 이벤트 메커니즘이 포함되어 있습니다.

이벤트 리스너 접근 방식을 통해 HTMX 이벤트에 쉽게 등록할 수 있습니다:

document.body.addEventListener("htmx:load", function (evt) {
  myJavascriptLib.init(evt.detail.elt);
});

HTMX 헬퍼:

htmx.on("htmx:load", function (evt) {
  myJavascriptLib.init(evt.detail.elt);
});

두 경우 모두 htmx:load 이벤트입니다.

htmx:load 이벤트는 HTMX가 DOM에 요소를 로드할 때마다 발생하며, 기능적으로 기존의 로드 이벤트와 동일합니다.

htmx:load 이벤트를 사용하여 타사 라이브러리를 초기화할 수도 있습니다:

htmx.onLoad(function (target) {
  myJavascriptLib.init(target);
});

이벤트를 사용하여 요청을 구성할 수도 있습니다. 이에 대한 예는 아래와 같습니다:

document.body.addEventListener("htmx:configRequest", function (evt) {
  evt.detail.parameters["auth_token"] = getAuthToken(); // add a new parameter into the request
  evt.detail.headers["Authentication-Token"] = getAuthToken(); // add a new header into the request
});

위 코드에서는 htmx:configRequest 이벤트를 사용하여 AJAX 요청이 전송되기 전에 변경했습니다:

htmx:beforeSwap 이벤트를 처리하여 HTMX의 스왑 동작을 조정할 수 있습니다.

document.body.addEventListener("htmx:beforeSwap", function (evt) {
  if (evt.detail.xhr.status === 404) {
    // alert the user when a 404 occurs (maybe use a nicer mechanism than alert())
    alert("Error: Could Not Find Resource");
  } else if (evt.detail.xhr.status === 422) {
    // allow 422 responses to swap as we are using this as a signal that
    // a form was submitted with bad data and want to rerender with the
    // errors
    //
    // set isError to false to avoid error logging in console
    evt.detail.shouldSwap = true;
    evt.detail.isError = false;
  } else if (evt.detail.xhr.status === 418) {
    // if the response code 418 (I'm a teapot) is returned, retarget the
    // content of the response to the element with the id `teapot`
    evt.detail.shouldSwap = true;
    evt.detail.target = htmx.find("#teapot");
  }
});

위 코드에서는 일반적으로 htmx에서 스왑이 발생하지 않는 400 수준의 오류 응답 코드를 몇 개 처리했습니다.

HTMX가 지원하는 더 많은 이벤트를 보려면 여기를 방문하세요.

로깅

HTMX는 htmx.logger 변수를 제공하며, 이 변수를 설정하면 모든 이벤트를 로깅합니다. 로거를 설정하는 방법에 대한 예제는 아래와 같습니다:

htmx.logger = function (elt, event, data) {
  if (console) {
    console.log(event, elt, data);
  }
};

결론

이 글에서는 HTMX 라이브러리에 대한 개요를 제공했습니다. HTMX를 사용하면 자바스크립트에 의존하지 않고도 Ajax 기능을 구현할 수 있어 동적 애플리케이션을 쉽게 구축할 수 있다는 것을 보여주었습니다. HTMX는 단순한 새로운 라이브러리 그 이상일 수 있습니다. 웹 개발에서 상호 작용에 접근하는 방식에 혁신을 가져올 수 있습니다.
