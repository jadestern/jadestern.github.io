상태 머신은 이벤트가 발생했을 때 프로세스가 상태에서 상태로 이동하는 방식을 모델링하는 데 도움이 됩니다.

상태 머신은 모든 상태, 이벤트 및 그 사이의 전환을 캡처하는 데 도움이 되기 때문에 소프트웨어 개발에서 유용합니다. 상태 머신을 사용하면 불가능한 상태를 쉽게 찾고 바람직하지 않은 전환을 발견할 수 있습니다.

상태 머신은 애플리케이션 로직을 모델링합니다. 아래는 비디오 플레이어의 로직입니다. 동영상이 재생되면 전체 화면 모드로 열립니다. 동영상이 중지되면 전체 화면 모드에서 닫힙니다. 동영상 플레이어가 전체 화면 모드일 때는 재생 중 또는 일시 중지 상태일 수 있습니다.

상태머신의 장점

  • 이해하기 쉽습니다. 스테이트 머신은 시각적이고 이해하기 쉽습니다. 팀 및 이해관계자와 소통하기 좋은 방법입니다.
  • 테스트가 간단합니다. 상태 머신은 결정론적이기 때문에 테스트가 간단합니다. 가능한 모든 상태와 상태 간 전환을 테스트할 수 있습니다.
  • 구현이 간단합니다. 상태 머신은 모든 프로그래밍 언어로 구현하기 쉽습니다. 라이브러리를 사용하거나 직접 작성할 수 있습니다.
  • 유지 관리가 간단합니다. 스테이트 머신은 이해, 테스트, 구현이 간단하기 때문에 유지 관리가 간단합니다.
  • 복잡한 사용자 인터페이스를 모델링하고 애플리케이션 상태를 관리할 수 있는 명확하고 간결한 방법을 제공합니다.
  • 상태 머신은 상태 전환을 관리하고 이벤트를 처리하는 구조화된 방법을 제공함으로써 복잡성을 줄이고 유지보수성을 개선할 수 있습니다.
  • 또한 애플리케이션의 여러 부분 간의 문제를 명확하게 분리하여 버그를 방지하고 코드 품질을 개선하는 데 도움이 될 수 있습니다.
  • 또한 상태 머신은 유지 관리가 용이하며 매우 복잡한 프론트엔드 프로세스를 훨씬 더 관리하기 쉽게 만들 수 있는 방법을 제공합니다.

스테이트차트란 무엇인가요?

상태 차트는 기존의 유한 상태 머신을 확장하여 보다 복잡한 로직을 모델링합니다.

상태차트는 순서도나 시퀀스 다이어그램처럼 상자와 화살표를 사용하는 상태 머신을 시각적으로 확장한 것입니다. 스테이트차트는 계층 구조, 동시성, 통신 등 일반 스테이트 머신에서는 사용할 수 없는 추가 기능을 추가합니다.

스테이트 스튜디오에서 스테이트 머신을 만들면 스테이트 차트도 스테이트 머신이 됩니다!

상태

상태는 머신의 상태 또는 모드를 설명하며, 잠자기 및 깨어남과 같은 간단한 상태일 수 있습니다. 스테이트 머신은 한 번에 한 가지 상태만 가질 수 있습니다.

스테이트먼트 스튜디오에서 강아지 상태 머신을 살펴보세요.

개는 항상 잠자거나 깨어 있습니다. 개는 동시에 잠자거나 깨어 있을 수 없으며, 개가 잠자지도 깨어 있지도 않은 상태일 수도 없습니다. 이 두 가지 상태, 즉 정확히 한정되고 유한한 수의 상태만 존재합니다.

상태 머신이 시작되면 가장 먼저 초기 상태로 들어갑니다. 머신은 최상위 초기 상태를 하나만 가질 수 있으며, 초기 상태가 여러 개 있으면 어디서부터 시작해야 할지 모릅니다!

개를 산책시키는 과정을 설명하는 이 상태 차트에서 초기 상태는 산책을 기다리고 있을 것입니다.

XState의 상태

상태는 states 속성 내부의 XState 상태 머신에서 정의됩니다.

import { createMachine } from 'xstate';

const dogMachine = createMachine({
  id: 'dog',
  initial: 'asleep',
  states: {
    asleep: {
      // ...
    },
    awake: {
      // ...
    },
    //...
  }
});

전환 및이벤트

개가 수면과 각성 사이를 오가는 방식은 트랜지션을 통해 이루어집니다. 이벤트는 트랜지션을 유발하고, 이벤트가 발생하면 머신은 다음 상태로 전환합니다. 개는 깨어나기 및 잠들기 이벤트를 통해 수면과 깨어남 사이를 오갑니다.

상태와 이벤트의 각 조합은 항상 동일한 다음 상태를 가리키며, 전환은 "결정론적"입니다. 개는 절대로 깨어났다가 다시 잠들거나 잠들었다가 다시 깨어나는 일이 없습니다.

스테이트먼트 스튜디오에서 이벤트와 트랜지션이 있는 개 상태 머신을 살펴보세요.

두 개의 유한 상태와 전환이 있는 이 작은 개 프로세스는 유한 상태 머신입니다. 상태 머신은 어떤 사물의 동작을 설명하는 데 사용됩니다. 이 기계는 사물의 상태와 그 상태 사이의 전환을 설명합니다. 유한 상태 머신은 상태의 수가 한정되어 있기 때문에 유한 상태 머신이라고 합니다. (전문 용어를 좋아하는 사람들은 FSM으로 줄여서 부르기도 합니다).

트랜지션과 이벤트에 대해 자세히 알아보세요.

XState의 이벤트 및 트랜지션

이벤트와 트랜지션은 상태의 on 속성 내부의 XState 상태 머신에서 정의됩니다.

import { createMachine } from 'xstate';

const dogMachine = createMachine({
  id: 'dog',
  initial: 'asleep',
  states: {
    asleep: {
      on: {
        'wakes up': 'awake',
      }
    },
    awake: {
      on: {
        'falls asleep': 'asleep',
      }
    },
    //...
  }
});

부모상태

부모 상태는 자식 상태라고도 하는 더 많은 상태를 포함할 수 있는 상태입니다. 이러한 자식 상태는 부모 상태가 발생하고 있을 때만 발생할 수 있습니다. 걷기 상태 안에는 걷기 및 달리기라는 자식 상태가 있을 수 있습니다.

부모 상태는 자식 상태를 담는 컨테이너 역할을 하는 레이블이 붙은 직사각형 상자로 표시됩니다. 부모 상태를 복합 상태라고도 합니다.

스테이트 스튜디오에서 개 산책기를 살펴보세요.

부모 상태는 어떤 자식 상태가 초기 상태인지도 지정해야 합니다. 산책 중 부모 상태에서는 초기 상태가 산책 중입니다.

부모 상태와 자식 상태는 스테이트차트가 일상적인 상태 머신보다 더 복잡한 문제를 처리할 수 있게 해주는 기능 중 하나입니다.

원자상태

원자 상태는 자식 상태가 없는 상태입니다. 대기 중, 걷기 완료, 걷기 및 달리기는 모두 원자 상태입니다.

병렬상태

병렬 상태는 영역이라고도 하는 모든 하위 상태가 동시에 활성화되어 있는 상태입니다. 영역은 병렬 상태 컨테이너 내부에서 파선으로 구분됩니다.

산책 중 평행 상태 내부에는 두 개의 영역이 있을 수 있습니다. 한 영역에는 개의 활동 하위 상태인 걷기 및 달리기, 다른 영역에는 개의 꼬리 상태인 흔들기 및 흔들지 않음이 포함됩니다. 개는 걷고 꼬리를 흔들거나 뛰고 꼬리를 흔들 수 있으며, 꼬리를 흔들지 않고 이 두 가지 활동을 모두 할 수도 있습니다.

스테이트 스튜디오에서 개 산책기를 살펴보세요.

두 영역 모두 어떤 자식 상태가 초기 상태인지 지정해야 합니다. 꼬리 영역에서는 초기 상태가 꼬리를 흔들지 않습니다.

자체 전환

자체 전환은 이벤트가 발생하지만 전환이 동일한 상태로 돌아가는 경우입니다. 전환 화살표는 동일한 상태를 종료했다가 다시 입력합니다.

자체 전환을 설명하는 유용한 방법은 프로세스에서 "어딘가로 이동하지 않고 무언가를 하는 것"입니다.

개가 구 걸하는 과정에서는 간식을 받는 이벤트와 함께 구걸하는 상태가 있을 것입니다. 간식을 좋아하는 개는 간식 받기 이벤트를 몇 번을 진행해도 다시 구걸 상태로 돌아갑니다.
