타입스크립트 세계에서는 할당 가능한 모든 값에 타입이 있습니다(지난 블로그에서 설명한 대로). 이제 특정 값의 유형을 식별하기 위해 유형 주석과 유형 추론이 있습니다.

1️⃣Type 어노테이션 -

타입 어노테이션의 핵심은 변수가 참조할 값의 유형을 타입스크립트에 알려주기 위해 작성하는 소량의 코드입니다.
따라서 개발자는 타입스크립트에 작업 중인 값의 유형을 알려줍니다.

타입 어노테이션의 예

변수를 사용한 타입 어노테이션

const numCars: number = 5;

:number 는 타입 어노테이션으로, 이제 타입스크립트에 숫자 유형만 numCars 변수에 할당할 것이며 다른 유형을 할당하려고 하면 오류가 빠르게 표시될 것임을 알려줍니다.

다른 타입을 사용하는 타입 어노테이션의 더 많은 예는 :

let speed: string = 'fast';
let isSuperCar: boolean = false;
let sunroof: null = null;
let nothing: undefined = undefined;
const todayDate: Date = new Date();

2️⃣Type 추론 -

타입 추론은 간단히 말해 타입스크립트가 값이 어떤 타입을 가질지 알아내려고 하는 방법입니다.
변수를 사용한 타입 추론의 일반적인 예는 다음과 같습니다:

let isSuperCar = false;

이제 부울이 아닌 값을 할당하려고 하면 오류가 표시됩니다.

오류에서 볼 수 있듯이 타입스크립트에서 isSuperCar가 어떤 타입을 가질지 알려주지 않았음에도 불구하고 isSuperCar 값이 어떤 타입을 가질지 식별했습니다.

3️⃣When 에서 타입 어노테이션과 타입 유추를 사용하세요.

타입 어노테이션을 사용해야 하는 경우:

  • 유형 추론이 불가능한 변수가 있을 때, 부울 또는 숫자 값을 가질 수 있는 변수가 있다고 가정해 보겠습니다. 이 경우 유형 추론만으로는 변수의 유형을 스스로 식별할 수 없으므로 다음과 같은 구문에서 유형 추론이 필요합니다:
//A car can either have one sunroof or none

let sunroof: boolean | number = false;

//Now if we assign it a value of 1, we will not get an error 

sunroof = 1;

//It will not give an error as we have annotated that it is going to have either a boolean or a number as a value

  • 한 줄에 값을 선언하고 나중에 초기화할 때,
    는 선언과 초기화가 같은 줄에 있기 때문에 이전의 유형 추론이 작동했습니다.

  • 이 경우 함수가 'any' 유형을 반환할 때는 미리 값을 명확히 해야 합니다.

유형 추론을 사용하는 경우 -

위의 코너 케이스를 제외하고는 항상 유형 추론에 의존하려고 노력합니다.
