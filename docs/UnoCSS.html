왜 UnoCSS인가?

동기

UnoCSS의 창시자인 Anthony Fu가 작성한 블로그 게시물 Reimagine Atomic CSS를 읽어보시면 UnoCSS의 동기를 더 잘 이해할 수 있을 것입니다.

UnoCSS는 X와 어떻게 다른가요?

윈디 CSS

UnoCSS는 Windi CSS의 팀원 중 한 명이 Windi CSS에서 수행한 작업에서 많은 영감을 얻어 시작했습니다. Windi CSS는 더 이상 활발하게 유지보수되지 않지만(2023년 3월 기준), UnoCSS는 Windi CSS의 '정신적 후계자 '라고 할 수 있습니다.

UnoCSS는 Windi CSS의 온디맨드 특성, 속성화 모드, 바로 가기, 변형 그룹, 컴파일 모드 등을 그대로 이어받았습니다. 또한 UnoCSS는 처음부터 최대한의 확장성과 성능을 염두에 두고 제작되었기 때문에 순수 CSS 아이콘, 값 없는 어트리뷰티파이, 태그화, 웹 글꼴 등과 같은 새로운 기능을 도입할 수 있습니다.

무엇보다도 UnoCSS는 모든 기능이 선택 사항인 원자적 CSS 엔진으로 추출되어 원하는 기능의 조합으로 나만의 규칙, 나만의 디자인 시스템, 나만의 프리셋을 쉽게 만들 수 있다는 점이 가장 큰 장점입니다.

테일윈드 CSS

Windi CSS와 UnoCSS는 모두 Tailwind CSS에서 많은 영감을 얻었습니다. UnoCSS는 기초부터 구축되었기 때문에 원자 CSS가 어떻게 선행 기술로 설계되고 우아하고 강력한 API로 추상화되었는지에 대한 훌륭한 개요를 파악할 수 있습니다. 디자인 목표가 상당히 다르기 때문에 Tailwind CSS와 사과와 사과를 비교하기는 어렵습니다. 하지만 몇 가지 차이점을 나열해 보겠습니다.

Tailwind CSS는 PostCSS 플러그인인 반면, UnoCSS는 빌드 도구( PostCSS 플러그인 포함)와의 최고급 통합 모음을 갖춘 동형 엔진입니다. 즉, UnoCSS는 훨씬 더 유연하게 여러 곳에서 사용할 수 있으며(예: 즉석에서 CSS를 생성하는 CDN 런타임), 빌드 도구와 긴밀하게 통합되어 더 나은 HMR, 성능 및 개발자 경험을 제공할 수 있습니다(예: 인스펙터).

기술적 장단점은 차치하고서라도, UnoCSS는 완전히 확장 가능하고 사용자 정의할 수 있도록 설계된 반면, Tailwind CSS는 좀 더 독단적입니다. Tailwind CSS 위에 사용자 정의 디자인 시스템(또는 디자인 토큰)을 구축하는 것은 어려울 수 있으며, Tailwind CSS의 관습에서 벗어날 수도 없습니다. UnoCSS를 사용하면 원하는 거의 모든 것을 완벽하게 제어하면서 구축할 수 있습니다. 예를 들어, 하나의 프리셋 안에 모든 Tailwind CSS 호환 유틸리티를 구현했으며, 다른 흥미로운 철학으로 구현된 멋진 커뮤니티 프리셋이 많이 있습니다.

UnoCSS가 제공하는 유연성 덕분에 그 위에 여러 가지 혁신적인 기능을 실험할 수 있었습니다:

  • 순수 CSS 아이콘
  • 속성 지정 모드
  • 변형 그룹
  • 단축키
  • 태그 지정
  • 웹 글꼴
  • CDN 런타임
  • 인스펙터

Tailwind CSS와 디자인 목표가 다르기 때문에 UnoCSS는 Tailwind CSS의 플러그인 시스템이나 구성을 지원하지 않으므로 고도로 사용자 정의된 Tailwind CSS 프로젝트에서 마이그레이션하기가 더 어려워질 수 있습니다. 이는 UnoCSS의 고성능과 확장성을 높이기 위한 의도된 결정이며, 그만한 가치가 있다고 생각합니다.
