소개

저는 Rust 프로그래밍 언어를 배우기로 결심했고, 그 방법은 다음과 같습니다.

저는 Rust에 관한 책을 읽는 것으로 시작했습니다. 시작하기에 좋은 방법이지만 그것만으로는 충분하지 않습니다. 책만 읽는다고 해서 프로그래밍을 배우는 것이 아니라 코드를 작성해야 하거든요. 그래서 저는 Rust로 간단한 비밀번호 관리자를 작성하기 시작했습니다. 어떻게 이 프로젝트 아이디어를 떠올렸는지는 정확히 모르겠지만, BitWarden이나 Dashlane과 같은 유료 앱을 여러 개 사용하고 있었는데 이를 대체할 수 있는 무언가를 만들어보고 싶었기 때문인 것 같습니다.

이 프로젝트를 위한 제 준비는 매우 기초적이었습니다. 책을 조금 읽긴 했지만 지식은 아주 미미했습니다. 그래도 많은 시행착오를 거쳐 프로젝트를 시작하고 몇 가지 기본 기능을 구현할 수 있었습니다. 인터넷에서 많은 것을 찾아보고 Rust 설명서를 읽어야 했습니다. 특히 Rust 차용 검사기 구문과 씨름을 많이 했습니다. 차용 검사기와 Rust의 메모리 안전 기능은 Rust를 배우기 어렵게 만들었습니다. 하지만 정말 놀라웠던 것은 마침내 프로그램을 컴파일했을 때 실제로 작동하고 제가 원하는 대로 작동한다는 것이었습니다. 디버깅이나 문제 해결을 할 필요 없이 그냥 작동하는 것이었습니다. 이 경험을 통해 Rust에 대해 매우 흥미를 갖게 되었습니다. 엄격하고 배우기 어려운 언어이지만 일단 코드를 컴파일하면 예상대로 작동할 가능성이 매우 높습니다.

Rust에 대한 기본적인 이해를 바탕으로 작동하는 간단한 비밀번호 관리자를 구현할 수 있었습니다. 저는 Passlane이라고 이름을 붙이고 Github에 공개했습니다. 제가 필요한 모든 기능을 갖추고 있었습니다: 비밀번호를 추가, 편집, 삭제할 수 있었습니다. 볼트에서 비밀번호를 검색하고 사용할 수 있었습니다. 새로운 비밀번호도 생성할 수 있었습니다. 사실 저는 이 프로그램을 Dashlane을 대체하는 용도로 사용하고 있었습니다.

이 시점에서 저는 몇 달 동안 Rust 학습을 중단하고 당시 더 관심이 있었던 다른 취미 프로젝트로 넘어갔습니다.

다시 Rust로 돌아오기

1년이 지난 후 다시 Rust로 돌아와서 계속 배우기로 결정했습니다. 이번에는 아마존에서 Rust에 대한 실제 책을 몇 권 구입했습니다. 이 책들로 무장하고 다시 읽고 배우기 시작했고 비밀번호 관리자 프로젝트도 다시 시작했습니다. 이전에 작성했던 지저분한 코드를 대대적으로 리팩토링하기 시작했습니다. Rust에 대해 더 많이 알게 되면서 코드베이스의 상당 부분을 교체했습니다. 특성, 수명, 오류 처리 및 기타 여러 개념에 대해 자세히 알아봤습니다. 새로운 지식을 습득할 때마다 비밀번호 관리자 코드베이스를 다시 살펴봤습니다.

비동기 프로그래밍에 대해 배웠을 때, CLI 기반 프로젝트에는 비동기가 실제로 필요하지 않다는 것을 깨달았습니다. 그래서 모든 비동기 코드와 Tokio 런타임을 제거하여 코드를 대폭 간소화했습니다.

오류 처리와 ? 연산자를 공부하면서 모든 unwrap() 호출을 ?로 대체했습니다. Result 유형을 수용하고 자체 오류 유형을 구현하여 코드베이스 전체에 사용했습니다. 이렇게 해서 코드베이스가 더 단순해졌고 일반적으로 단순할수록 더 좋습니다.

또한 Rust 모듈 시스템을 연구하고 모듈과 훨씬 작은 파일을 사용하도록 코드베이스를 리팩터링했습니다. 이것도 또 다른 개선이었습니다.

새로운 지식을 공부한 다음 프로젝트의 상당히 큰 코드베이스에 즉시 적용하는 이 과정이 제 학습 과정의 핵심이었습니다. 이 접근 방식은 제 이해를 더욱 확고히 해주었습니다. 제 생각에는 프로그래밍 언어를 배우는 가장 효과적인 방법이라고 생각합니다.

프로젝트의 커밋 기록에서 저의 광범위한 재작성 및 리팩터링을 확인할 수 있습니다. 또한 앱 2.3.0 버전에서 스토리지 백엔드로 Keepass로 전환하여 기능을 크게 변경했습니다. 사용자가 데이터를 안전하게 보관하기 위해 올바른 조치를 취하고 있다는 것을 신뢰할 수 있도록 하기 위해 변경했습니다. 또한 2단계 인증을 위해 TOTP 코드(시간 제한 일회용 비밀번호)에 대한 지원을 추가했습니다.

결론

결론은 이것입니다: 프로그래밍을 배울 때는 가능한 한 많은 코드를 작성해야 합니다. 동시에 책과 문서를 읽으세요. 인공지능과 챗봇을 사용하여 질문하세요. Copilot이나 ChatGPT와 같은 도구에 도움을 요청하는 것도 매우 효과적인 학습 방법이 될 수 있지만, 이는 다른 블로그 게시물에서 다룰 주제입니다.

프로젝트

저는 Github에서 비밀번호 관리자를 사용하고 있습니다. Passlane이라는 이름의 이 프로젝트는 현재로서는 꽤 자랑스럽습니다. 확인해 보시고 어떻게 생각하시는지 알려주세요!
