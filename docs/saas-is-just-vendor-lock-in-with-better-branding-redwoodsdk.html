SaaS는 더 나은 브랜딩으로 공급업체에 종속되는 것일 뿐입니다 | RedwoodSDK

개발자는 "제품에 집중하라"는 말을 듣고 나머지는 SaaS 공급업체에 맡기지만 인증, 대기열, 파일 저장, 이미지 최적화 등 타사 서비스를 통합하는 데는 비용이 듭니다. 비용뿐만 아니라 시간, 마찰, 정신적 오버헤드도 발생합니다.

SaaS를 스택에 통합할 때마다 지불해야 하는 5가지 숨겨진 세금이 있습니다.

1. 발견세

통합하기 전에 먼저 실제로 무엇을 판매하고 있는지 파악해야 하나요?

  1. 어떤 문제를 해결하고 있나요?
  2. 스택과 호환되는가?
  3. 여러분의 규모에 맞는 합리적인 가격인가요?
  4. 문서가 명확하고 구현상 이상한 점은 없는가?

이 무급 연구 작업은 일반적으로 양도할 수 없습니다. '업로더'나 '메가큐'에 대해 배운 내용은 다음에 다른 것을 평가할 때 도움이 되지 않습니다. 또한 주관적입니다. 마케팅인데 마케팅 메시지가 공감되나요?

2. 가입비

서비스를 결정하고 이메일과 신용카드를 넘겨주는 순간입니다.

  1. 사용량 기반 요금제를 지원하나요, 아니면 락인 티어만 지원하나요?
  2. 팀원들이 대시보드에 액세스할 수 있나요, 아니면 해당 기능에 대해 더 많은 비용을 지불해야 하나요? 서비스를 같은 양만 사용하는데도 불구하고!
  3. 페이월을 거치지 않고 제품을 테스트할 수 있나요?

코드를 한 줄도 작성하지 않았더라도 이제 바로 사용할 수 있습니다.

3. 통합 세금

이제 본격적인 작업이 시작됩니다.

  1. 문서를 읽으세요.
  2. 라이브러리를 설치합니다.
  3. 프레임워크에 연결합니다.
  4. 그리고 문서에 언급되지 않은 에지 케이스를 파악하세요. 문서가 곧 마케팅이기 때문입니다!

종종 여러분은 자신만의 도구와 싸우게 됩니다. 그들은 가장 낮은 공통 분모를 목표로 하고 있고, 여러분은 최첨단을 지향하고 있습니다. 또는 그 반대일 수도 있습니다!

4. 지역 개발세

로컬에서 작업하려면 SaaS 서비스가 필요합니다. 로컬 에뮬레이터도 제공하나요? 테스트에 사용할 수 있나요? 하나의 기능을 테스트하기 위해 클라우드로 터널링해야 하나요?

이제 프로덕션용, 스테이징용, 로컬용... 운이 좋다면 구성 로직을 분기할 수 있습니다.

5. 생산 세금

이 부분은 "끝난" 부분이지만, 그렇지 않은 부분도 있습니다.

  1. 스테이징 환경에서 이 기능을 사용할 수 있나요? 풀 리퀘스트 미리보기는 어떤가요?
  2. API 키를 안전하게 관리해야 합니다.
  3. 모니터링, 로깅 및 알림
  4. 노트북에서는 작동하는데 왜 프로덕션에서는 실패하는지 궁금하신가요?

서비스를 통합했지만 이제 프로덕션 환경에서 안정적으로 작동하는지 확인해야 합니다.

결론

최신 SaaS의 핵심은 "바퀴를 재발명하지 말라"는 것입니다. 하지만 모든 바퀴에는 마찰이 따르기 마련입니다. 단순한 서비스가 아닙니다: 계약입니다. 종속성입니다. 이는 미묘한 구조적 변화이며 세금과 함께 제공됩니다.

어떤 선택을 하든 항상 공급업체에 종속될 수밖에 없습니다. 오픈 소스이고 자체 호스팅이라 하더라도 무언가를 전환한다는 것은 많은 코드를 다시 작성해야 한다는 것을 의미합니다.

따라서 저는 그런 결정을 내리지 말라고 주장합니다. 그냥 플랫폼을 선택하세요. 중요한 것은 프레임워크나 실행되는 서비스가 아니라 작성하고자 하는 소프트웨어입니다.

Cloudflare나 Supabase 같은 플랫폼이 빛을 발합니다. 데이터베이스, 큐, 이미지 서비스, 스토리지가 모두 같은 플랫폼에 있고 같은 언어를 사용하는 곳입니다. 이러한 세금을 반복적으로 납부하지 않아도 됩니다. 이미 있는 제품을 선택하기만 하면 됩니다.

  1. 공급업체 간에 컨텍스트 전환이 필요 없습니다.
  2. API 키 랭글링이 필요 없습니다.
  3. 호환성 해킹이나 설정 포크도 없습니다.
  4. 개발팀과 프로덕션팀에서 동일하게 작동하는 빠르고 로컬한 느낌의 통합만 있으면 됩니다.

모든 것이 동일한 시스템에서 실행되는 것처럼 느껴지며, 어떤 면에서는 실제로도 그렇습니다. 이것이 바로 통합 플랫폼의 숨겨진 강점입니다. 통합 플랫폼은 코드와 서비스 사이의 거리를 좁혀줍니다. 그리고 이를 통해 어떤 SaaS 공급업체도 판매할 수 없는 한 가지를 고객에게 돌려줍니다: 바로 "흐름"입니다.
