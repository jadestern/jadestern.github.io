프로덕션 환경에서 사용하기 전에 브라우저 호환성 표를 주의 깊게 확인하세요.

CSS 그리드 레이아웃 사양의 레벨 3에는 masonry 값이 포함되어 있습니다. grid-template-columns 및 grid-template-rows. 이 가이드에서는 벽돌식 레이아웃의 정의와 사용 방법을 자세히 설명합니다.

벽돌식 레이아웃은 한 축은 일반적인 엄격한 그리드 레이아웃(대부분 열)을 사용하고 다른 축은 벽돌식 레이아웃을 사용하는 레이아웃 방식입니다. 벽돌식 축에서는 짧은 항목 뒤에 간격이 남는 엄격한 격자를 고수하는 대신 다음 행의 항목이 위로 올라와서 간격을 완전히 채웁니다.

조적 레이아웃 만들기

가장 일반적인 벽돌식 레이아웃을 만들려면 열이 그리드 축이 되고 행이 벽돌식 축이 됩니다. grid-template-columns 및 grid-template-rows 으로 이 레이아웃을 정의합니다:

.container {
  display: grid;
  gap: 10px;
  grid-template-columns: repeat(auto-fill, minmax(120px, 1fr));
  grid-template-rows: masonry;
}

이제 이 컨테이너의 자식 요소는 일반 그리드 레이아웃 자동 배치와 마찬가지로 행을 따라 항목별로 배치됩니다. 그러나 새 행으로 이동하면 항목은 벽돌 알고리즘에 따라 표시됩니다. 항목은 공간이 가장 많은 열에 로드되어 엄격한 행 트랙 없이 꽉 찬 레이아웃이 생성됩니다.

항목이 행에 로드되는 매조 레이아웃을 만들 수도 있습니다.

그리드 축 제어하기

그리드 축에서는 그리드 레이아웃에서 예상한 대로 항목이 작동합니다. span 키워드를 사용하여 자동 배치 상태를 유지하면서 항목이 여러 트랙에 걸쳐 있도록 할 수 있습니다. 라인 기반 위치 지정을 사용하여 항목을 배치할 수도 있습니다.

스팬 항목이 있는 벽돌식 레이아웃

이 예제에서는 두 개의 항목이 두 개의 트랙에 걸쳐 있으며, 석조물 항목이 그 주위를 둘러싸고 작동합니다.

이 예시에는 열에 대한 위치 지정이 있는 항목이 포함되어 있습니다. 명확한 위치가 있는 항목은 벽돌식 레이아웃이 발생하기 전에 배치됩니다.

폴백

매소리를 지원하지 않는 브라우저에서는 일반 그리드 자동 배치가 대신 사용됩니다.

브라우저 호환성

css.properties.grid-template-columns.masonry

BCD 테이블은 브라우저에서만 로드됩니다.

css.properties.grid-template-rows.masonry

BCD 테이블은 브라우저에서만 로드됩니다.
