모나코 에디터

모나코 에디터는 VS Code의 모든 기능을 갖춘 코드 편집기입니다. 지원되는 기능 중 일부를 확인하려면 VS Code 문서를 확인하세요.

사용해 보기

인터랙티브 플레이그라운드에서 에디터를 사용해 보고 다양한 예제를 살펴보세요.

플레이그라운드는 에디터 사용 방법, 지원되는 기능, 다양한 버전을 사용해 보고 버그 보고를 위한 최소한의 재현 가능한 예제를 만들 수 있는 가장 좋은 방법입니다.

설치

> npm install monaco-editor

다음을 얻을 수 있습니다:

  • 내부 /esm: ESM 버전의 편집기(예: 웹팩과 호환)
  • 내부 /dev: AMD 번들, 축소되지 않음
  • 내부 /min: AMD 번들 및 축소 버전
  • /min-maps 내부 : 소스 맵 min
  • monaco.d.ts에디터의 API를 지정합니다(실제로 버전이 지정된 것, 그 외의 모든 것은 비공개로 간주되며 어떤 릴리스에서든 깨질 수 있음).

dev 버전을 기준으로 개발할 것을 권장하며, 프로덕션 환경에서는 min 버전을 사용하는 것이 좋습니다.

개념

모나코 에디터는 VS Code를 구동하는 텍스트 에디터로 가장 잘 알려져 있습니다. 하지만 조금 더 미묘한 차이가 있습니다. 모나코 에디터를 효과적으로 사용하려면 기본 개념에 대한 기본적인 이해가 필요합니다.

모델

모델은 모나코 에디터의 핵심입니다. 콘텐츠를 관리할 때 상호 작용하는 것이 바로 모델입니다. 모델은 열린 파일을 나타냅니다. 파일 시스템에 존재하는 파일을 나타낼 수도 있지만 반드시 그럴 필요는 없습니다. 예를 들어 모델은 텍스트 콘텐츠를 보관하고, 콘텐츠의 언어를 결정하며, 콘텐츠의 편집 기록을 추적합니다.

URI

각 모델은 URI로 식별됩니다. 그렇기 때문에 두 모델이 동일한 URI를 가질 수 없습니다. 모나코 편집기에서 콘텐츠를 표현할 때는 사용자가 편집하는 파일과 일치하는 가상 파일 시스템을 사용하는 것이 이상적입니다. 예를 들어 file:/// 을 기본 경로로 사용할 수 있습니다. URI 없이 모델을 생성하는 경우 해당 모델의 URI는 inmemory://model/1 입니다. 모델이 더 많이 생성되면 이 숫자는 증가합니다.

편집자

편집기는 사용자가 보는 모델 보기입니다. 이것은 DOM에 첨부되고 사용자에게 시각적으로 표시되는 것입니다. 일반적인 편집기 작업은 모델을 표시하거나, 보기 상태를 관리하거나, 작업 또는 명령을 실행하는 것입니다.

공급자

공급자는 스마트 편집기 기능을 제공합니다. 예를 들어 여기에는 완료 및 호버 정보가 포함됩니다. 이는 언어 서버 프로토콜 기능과 동일하지는 않지만 종종 매핑됩니다.

제공업체는 모델에서 작업합니다. 일부 스마트 기능은 파일 URI에 따라 달라집니다. 예를 들어 TypeScript가 가져오기를 해결하거나 JSON IntelliSense가 어떤 JSON 스키마를 어떤 모델에 적용할지 결정하는 것이 그 예입니다. 따라서 적절한 모델 URI를 선택하는 것이 중요합니다.

일회용품

많은 모나코 관련 개체는 종종 .dispose() 메서드를 구현합니다. 이 메서드는 리소스가 더 이상 필요하지 않을 때 정리를 수행하기 위한 것입니다. 예를 들어 model.dispose() 을 호출하면 등록이 취소되어 새 모델을 위한 URI를 확보할 수 있습니다. 편집자는 리소스를 확보하고 모델 리스너를 제거해야 합니다.

FAQ

VS 코드와 모나코 에디터의 관계는 어떻게 되나요?

모나코 에디터는 코드가 외부 웹 브라우저에서 실행되는 데 필요한 서비스를 중심으로 약간의 심을 추가하여 VS Code의 소스에서 바로 생성됩니다.

VS Code의 버전과 모나코 에디터의 버전은 어떤 관계가 있나요?

관계 없습니다. 모나코 에디터는 라이브러리이며 소스 코드를 직접 반영합니다.

VS Code용 확장 프로그램을 작성했는데 브라우저의 모나코 에디터에서 작동하나요?

아니요.

참고: 확장 프로그램이 LSP를 완전히 기반으로 하고 언어 서버가 자바스크립트로 작성된 경우에는 가능합니다.

이 모든 웹 워커가 왜 필요하며 왜 신경 써야 하나요?

언어 서비스는 UI 스레드 외부에서 무거운 작업을 계산하기 위해 웹 워커를 생성합니다. 리소스 오버헤드 측면에서 비용이 거의 들지 않으며, 제대로 작동하기만 한다면 크게 걱정할 필요가 없습니다(위의 크로스 도메인 사례 참조).

loader.js? require.js 을 사용할 수 있나요?

VS Code에서 사용하는 AMD 로더입니다. 예.

"웹 워커를 만들 수 없습니다"라는 경고가 표시됩니다. 어떻게 해야 하나요?

HTML5는 file:// 에 로드된 페이지가 웹 워커를 생성하는 것을 허용하지 않습니다. http:// 또는 https:// 스키마에서 웹 서버로 에디터를 로드하세요.

모바일 브라우저 또는 모바일 웹 앱 프레임워크에서 편집기가 지원되나요?

아니요.

에디터가 텍스트메이트 문법을 지원하지 않는 이유는 무엇인가요?

  • 에디터에서 TM 문법을 지원하려면 monaco-editor, vscode-oniguruma 및 vscode-textmate 을 통합한 https://github.com/bolinfest/monaco-tm 을 참조하세요.
