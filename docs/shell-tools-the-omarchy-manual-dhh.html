fzf

fzf를 사용하면 ff 별칭을 통해 파일을 퍼지 방식으로 찾을 수 있습니다. 아무 디렉터리로 이동하여 ff 을 입력하면 오른쪽에서 범위를 좁히고 있는 파일의 미리 보기를 보면서 해당 트리의 모든 파일로 퍼지 검색을 할 수 있습니다.

Ctrl + R 에서 fzf를 사용하여 명령 기록을 통해 퍼지 찾기를 할 수 있습니다.

이 도구는 Neovim에서 Space Space 을 입력할 때도 사용됩니다.

Zoxide

Zoxide는 cd를 대체하는 도구입니다. 이 도구는 사용자가 들어갔던 디렉터리를 기억하므로 다음에 더 쉽게 해당 디렉토리로 이동할 수 있습니다. cd ~/.local/share/omarchy 을 한 번 입력했다고 가정해 보겠습니다. 다음에는 cd omarchy (또는 cd oma)을 입력하면 Zoxide가 바로 해당 디렉토리로 이동합니다.

ripgrep

ripgrep는 rg <pattern> <path> 를 사용하여 파일의 내용을 검색합니다(예: rg Controller app/ 을 사용하여 디렉토리 앱에서 Controller 의 모든 멘션을 찾습니다).

이 도구는 Neovim에서 Space S G 을 입력할 때도 사용됩니다.

eza

eza는 ls를 대체합니다. 더 많은 정보, 색상 및 아이콘이 포함된 디렉토리 목록을 제공합니다. 기본적으로 eza는 ls로 별칭이 지정되어 있습니다. lt 을 사용하여 2단계 중첩 목록을 얻을 수도 있습니다. lsa 은 숨겨진 파일을 포함한 목록을 제공합니다. lta 은 숨겨진 파일이 포함된 중첩 목록을 제공합니다.

fd

fd는 find 를 대체하여 사용하기 쉬운 방법입니다. fd person.rb 을 사용하면 현재 트리 내에서 person.rb 이라는 파일을 찾을 수 있습니다. fd person.rb / 은 전체 파일 시스템을 검색합니다. fd person.rb / -H 은 숨겨진 디렉터리를 포함한 전체 파일 시스템을 검색합니다.
