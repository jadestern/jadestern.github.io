Anthropic 팀이 Claude Code를 사용하는 방법

앤트로픽의 내부 팀은 Claude Code로 워크플로를 혁신하여 개발자와 비기술 직원이 복잡한 프로젝트를 처리하고, 작업을 자동화하고, 이전에는 생산성을 제한했던 기술 격차를 해소할 수 있도록 지원하고 있습니다.

자세한 내용을 알아보기 위해 다음 팀과 이야기를 나누었습니다:

  • 데이터 인프라
  • 제품 개발
  • 보안 엔지니어링
  • 추론
  • 데이터 과학 및 시각화
  • 제품 엔지니어링
  • 성장 마케팅
  • 제품 디자인
  • 강화 학습(RL) 엔지니어링
  • 법률

이 인터뷰를 통해 다양한 부서에서 Claude Code를 사용하는 방법, 업무에 미치는 영향, 도입을 고려 중인 다른 조직을 위한 팁에 대한 인사이트를 수집했습니다.

데이터 인프라스트럭처를 위한 클로드 코드

데이터 인프라 팀은 회사 전체의 팀을 위해 모든 비즈니스 데이터를 정리합니다. 이들은 일상적인 데이터 엔지니어링 작업을 자동화하고, 복잡한 인프라 문제를 해결하며, 기술 및 비기술 팀원들이 독립적으로 데이터에 액세스하고 조작할 수 있도록 문서화된 워크플로를 만드는 데 Claude Code를 사용합니다.

주요 Claude Code 사용 사례

스크린샷을 사용한 Kubernetes 디버깅

Kubernetes 클러스터가 다운되어 새 파드를 예약하지 못했을 때, 팀은 Claude Code를 사용하여 문제를 진단했습니다. 대시보드의 스크린샷을 Claude Code에 입력하면, 대시보드의 메뉴별로 Google Cloud의 UI를 안내하여 포드 IP 주소 고갈을 나타내는 경고를 발견할 때까지 안내해 주었습니다. 그런 다음 Claude Code는 네트워킹 전문가의 개입 없이도 새 IP 풀을 생성하고 클러스터에 추가하는 정확한 명령을 제공했습니다.

재무팀을 위한 일반 텍스트 워크플로우

엔지니어들은 재무 팀원들에게 데이터 워크플로우를 설명하는 일반 텍스트 파일을 작성한 다음 이를 Claude Code에 로드하여 완전히 자동화된 실행을 얻는 방법을 보여주었습니다. 코딩 경험이 없는 직원도 '이 대시보드 쿼리, 정보 가져오기, 이 쿼리 실행, Excel 출력 생성'과 같은 단계를 설명하면 Claude Code가 날짜 같은 필수 입력을 요청하는 등 전체 워크플로우를 실행합니다.

신입 사원을 위한 코드베이스 탐색

새로운 데이터 과학자가 팀에 합류하면 Claude Code를 사용하여 방대한 코드베이스를 탐색하도록 안내받습니다. Claude Code는 Claude.md 파일(문서)을 읽고, 특정 작업과 관련된 파일을 식별하고, 데이터 파이프라인 종속성을 설명하고, 대시보드에 어떤 업스트림 소스를 제공하는지 신규 입사자가 이해할 수 있도록 도와줍니다. 이는 기존의 데이터 카탈로그와 검색 도구를 대체합니다.

세션 종료 후 문서 업데이트

팀은 각 작업이 끝날 때마다 완료된 작업 세션을 요약하고 개선 사항을 제안하도록 Claude Code에게 요청합니다. 이를 통해 지속적인 개선 루프가 만들어지고, Claude Code는 실제 사용법을 기반으로 Claude.md 문서와 워크플로 지침을 개선하여 후속 반복을 더욱 효과적으로 수행할 수 있도록 돕습니다.

여러 인스턴스에서 병렬 작업 관리

장기적인 데이터 작업을 수행할 때 팀은 프로젝트마다 다른 리포지토리에 여러 개의 Claude Code 인스턴스를 엽니다. 각 인스턴스는 전체 컨텍스트를 유지하므로 몇 시간 또는 며칠 후에 다시 전환해도 Claude Code는 작업 중이던 내용과 중단한 부분을 정확히 기억하므로 컨텍스트 손실 없이 진정한 병렬 워크플로우 관리가 가능합니다.

팀에 미치는 영향

전문 지식 없이도 인프라 문제 해결

일반적으로 시스템 또는 네트워킹 팀원을 투입해야 하는 Kubernetes 클러스터 문제를 Claude Code를 사용하여 문제를 진단하고 정확한 수정 사항을 제공함으로써 해결합니다.

온보딩 가속화

새로운 데이터 분석가와 팀원들은 광범위한 안내 없이도 복잡한 시스템을 빠르게 이해하고 의미 있는 기여를 할 수 있습니다.

향상된 지원 워크플로

Claude Code는 훨씬 더 많은 양의 데이터를 처리하고 사람이 수동으로 검토하기에는 불가능한 이상 징후(예: 200개의 대시보드 모니터링)를 식별할 수 있습니다.

팀 간 셀프 서비스 지원

코딩 경험이 없는 재무팀도 이제 복잡한 데이터 워크플로를 독립적으로 실행할 수 있습니다.

데이터 인프라 팀의 유용한 팁

상세한 Claude.md 파일 작성

팀에 따르면 워크플로, 도구 및 기대치를 Claude.md 파일에 더 잘 문서화할수록 Claude Code의 성능이 더 좋아진다고 합니다. 따라서 기존 디자인 패턴이 있을 때 새로운 데이터 파이프라인을 설정하는 것과 같은 일상적인 작업에서 Claude Code가 뛰어난 성능을 발휘합니다.

민감한 데이터에는 CLI 대신 MCP 서버 사용

특히 로깅이 필요하거나 잠재적인 개인정보 보호 문제가 있는 민감한 데이터를 처리하는 경우, Claude Code가 액세스할 수 있는 항목에 대한 보안 제어를 강화하기 위해 BigQuery CLI 대신 MCP 서버를 사용할 것을 권장합니다.

팀 사용 세션 공유

팀은 팀원들이 서로에게 Claude Code 워크플로우를 시연하는 세션을 가졌습니다. 이를 통해 모범 사례를 전파하고 스스로 발견하지 못했을 수도 있는 다양한 도구 사용 방법을 보여주었습니다.

제품 개발을 위한 클로드 코드

클로드 코드 제품 개발 팀은 자체 제품을 사용하여 클로드 코드의 업데이트를 구축하여 제품의 엔터프라이즈 기능과 에이전트 루프 기능을 확장합니다.

주요 Claude Code 사용 사례

자동 승인 모드를 통한 빠른 프로토타이핑

엔지니어는 '자동 수락 모드'(Shift+Tab)를 활성화하고 Claude가 코드를 작성하고 테스트를 실행하며 지속적으로 반복하는 자율 루프를 설정하여 신속한 프로토타이핑을 위해 Claude Code를 사용합니다. Claude에게 익숙하지 않은 추상적인 문제를 주고 자율적으로 작업하게 한 다음 80% 완성된 솔루션을 검토한 후 최종 수정 작업을 맡깁니다. 팀은 깨끗한 git 상태에서 시작하고 정기적으로 체크포인트를 커밋하여 Claude가 잘못된 변경 사항을 쉽게 되돌릴 수 있도록 제안합니다.

핵심 기능을 위한 동기식 코딩

애플리케이션의 비즈니스 로직에 영향을 미치는 보다 중요한 기능의 경우, 팀은 Claude Code와 동시에 작업하면서 구체적인 구현 지침과 함께 자세한 프롬프트를 제공합니다. 팀은 실시간으로 프로세스를 모니터링하여 코드 품질, 스타일 가이드 준수 및 적절한 아키텍처를 보장하는 동시에 반복적인 코딩 작업은 Claude가 처리하도록 합니다.

Vim 모드 구축

가장 성공적인 비동기 프로젝트 중 하나는 클로드 코드에 Vim 키 바인딩을 구현하는 것이었습니다. 그들은 Claude에게 전체 기능을 구축하도록 요청했고, 최종 구현의 약 70%가 Claude의 자율적인 작업으로 이루어졌으며, 몇 번의 반복 작업만으로 완료할 수 있었습니다.

테스트 생성 및 버그 수정

팀은 기능을 구현한 후 종합적인 테스트를 작성하고 풀 리퀘스트 검토에서 확인된 간단한 버그 수정을 처리하기 위해 Claude Code를 사용합니다. 또한 GitHub Actions를 사용하여 서식 문제나 함수 이름 변경과 같은 풀 리퀘스트 코멘트를 자동으로 처리하도록 Claude를 설정합니다.

코드베이스 탐색

익숙하지 않은 코드베이스(예: 모노레포 또는 API 쪽)로 작업할 때 팀은 Claude Code를 사용하여 시스템의 작동 방식을 빠르게 이해합니다. Slack의 응답을 기다리는 대신 Claude에게 직접 설명과 코드 참조를 요청하여 컨텍스트 전환에 소요되는 시간을 크게 절약할 수 있습니다.

팀에 미치는 영향

더 빠른 기능 구현

Claude Code는 코드의 70%를 Claude가 자율적으로 작성하여 Vim 모드와 같은 복잡한 기능을 성공적으로 구현했습니다.

개발 속도 향상

이 도구는 구현 세부 사항에 얽매이지 않고 신속하게 기능을 프로토타이핑하고 아이디어를 반복합니다.

자동화된 테스트를 통한 코드 품질 향상

Claude는 포괄적인 테스트를 생성하고 일상적인 버그 수정을 처리하여 높은 표준을 유지하면서 수작업을 줄여줍니다.

더 나은 코드베이스 탐색

팀원들은 동료의 응답을 기다릴 필요 없이 모노레포의 익숙하지 않은 부분을 빠르게 파악할 수 있습니다.

Claude Code 팀의 유용한 팁

자급자족 루프 생성

빌드, 테스트 및 린트를 자동으로 실행하여 자체적으로 작업을 검증하도록 Claude를 설정하세요. 이렇게 하면 Claude가 더 오래 자율적으로 작업하고 스스로 실수를 찾아낼 수 있으며, 특히 코드를 작성하기 전에 Claude에게 테스트를 생성하도록 요청할 때 효과적입니다.

작업 분류 직관력 개발

비동기식으로 잘 작동하는 작업(주변 기능, 프로토타이핑)과 동기식 감독이 필요한 작업(핵심 비즈니스 로직, 중요한 수정 사항)을 구분하는 방법을 배워보세요. 제품 가장자리에 있는 추상적인 작업은 '자동 승인 모드'로 처리할 수 있지만, 핵심 기능은 면밀한 감독이 필요합니다.

명확하고 상세한 프롬프트 양식

컴포넌트의 이름이나 기능이 비슷한 경우에는 요청을 매우 구체적으로 작성하세요. 프롬프트가 명확하고 상세할수록 코드베이스의 잘못된 부분을 예기치 않게 변경하지 않고 독립적으로 작동하는 Claude를 더욱 신뢰할 수 있습니다.

보안 엔지니어링을 위한 Claude 코드

보안 엔지니어링 팀은 소프트웨어 개발 수명 주기, 공급망 보안 및 개발 환경 보안을 보호하는 데 중점을 둡니다. 이들은 코드 작성 및 디버깅에 Claude Code를 광범위하게 사용합니다.

주요 Claude Code 사용 사례

복잡한 인프라 디버깅

인시던트 작업 시 Claude Code 스택 추적 및 문서를 제공하여 코드베이스를 통해 제어 흐름을 추적하도록 요청합니다. 이렇게 하면 프로덕션 문제 해결 시간이 크게 단축되어 일반적으로 수동 코드 스캔에 10~15분이 걸리는 문제를 약 5분 만에 파악할 수 있습니다.

Terraform 코드 검토 및 분석

보안 승인이 필요한 인프라 변경의 경우, 팀은 Terraform 계획을 Claude Code에 복사하여 "이렇게 하면 어떻게 될까? 후회하게 될까?"라는 질문을 던집니다. 이렇게 하면 피드백 루프가 더욱 긴밀해지고 보안 팀이 인프라 변경 사항을 신속하게 검토하고 승인하기가 쉬워져 개발 프로세스의 병목 현상이 줄어듭니다.

문서 종합 및 런북

Claude Code는 여러 문서 소스를 수집하여 마크다운 런북, 문제 해결 가이드 및 개요를 만듭니다. 팀은 이러한 압축된 문서를 실제 문제를 디버깅하기 위한 컨텍스트로 사용하여 전체 지식 베이스를 검색하는 것보다 더 효율적인 워크플로우를 만듭니다.

테스트 중심 개발 워크플로

이전의 '디자인 문서 → 엉성한 코드 → 리팩터링 → 테스트 포기' 패턴 대신 이제 Claude Code에 의사 코드를 요청하고 테스트 중심 개발을 안내하며 막힐 때 주기적으로 체크인하여 조정함으로써 더 안정적이고 테스트 가능한 코드가 만들어집니다.

컨텍스트 전환 및 프로젝트 온보딩

"종속"(보안 승인 워크플로우를 위한 웹 애플리케이션)과 같은 기존 프로젝트에 기여할 때 Claude Code를 사용하여 마크다운으로 작성되고 코드베이스에 저장된 사양을 작성, 검토 및 실행하므로 몇 주가 아닌 며칠 내에 의미 있는 기여를 할 수 있습니다.

팀에 미치는 영향

인시던트 해결 시간 단축

일반적으로 수동 코드 스캔으로 10~15분이 걸리던 인프라 디버깅이 이제 5분 정도면 완료됩니다.

보안 검토 주기 개선

보안 승인을 위한 Terraform 코드 검토가 훨씬 더 빠르게 진행되므로 보안 팀의 승인을 기다리는 동안 개발자가 차단되는 일이 사라집니다.

부서 간 기여도 향상

팀원들은 컨텍스트 구축에 몇 주가 걸리는 대신 며칠 내에 프로젝트에 의미 있게 기여할 수 있습니다.

문서화 워크플로 개선

여러 소스에서 종합된 문제 해결 가이드와 런북을 통해 보다 효율적인 디버깅 프로세스를 만들 수 있습니다.

원샷으로 먼저 시도한 후 공동 작업하기

Claude에게 간단한 프롬프트를 주고 전체 구현을 먼저 시도하게 하세요. 성공하면(약 3분의 1 정도) 상당한 시간을 절약한 것입니다. 그렇지 않다면 보다 협업적이고 안내에 따른 접근 방식으로 전환하세요.

법무팀을 위한 클로드 코드

법무팀은 실험을 통해 클로드 코드의 잠재력을 발견하고 Anthropic의 제품에 대해 배우고자 하는 열망을 가졌습니다. 또한 한 팀원은 가족 및 업무용 프로토타입을 위한 접근성 도구 제작과 관련된 개인적인 사용 사례를 통해 비개발자를 위한 기술의 힘을 보여주었습니다.

주요 클로드 코드 사용 사례

가족 구성원을 위한 맞춤형 접근성 솔루션

팀원들은 의학적 진단으로 인해 언어 장애가 있는 가족 구성원을 위한 의사소통 보조 도구를 만들었습니다. 한 개인이 단 한 시간 만에 음성 뱅크를 사용하여 답변을 제안하고 음성으로 말하는 네이티브 음성 텍스트 변환을 사용하는 예측 텍스트 앱을 만들어 언어 치료사가 추천하는 기존 접근성 도구의 공백을 해결했습니다.

법무 부서 워크플로 자동화

팀은 팀원들이 Anthropic에서 적합한 변호사와 연결될 수 있도록 프로토타입 '전화 트리' 시스템을 만들어 법무 부서에서 기존의 개발 리소스 없이도 일반적인 작업을 위한 맞춤형 도구를 구축할 수 있는 방법을 보여주었습니다.

팀 조정 도구

관리자는 매주 팀 업데이트를 자동화하고 제품 전반의 법률 검토 상태를 추적하는 G Suite 애플리케이션을 구축하여 변호사가 스프레드시트 관리 대신 간단한 버튼 클릭으로 검토가 필요한 항목에 신속하게 플래그를 지정할 수 있도록 했습니다.

솔루션 검증을 위한 신속한 프로토타이핑

Claude Code를 사용하여 도메인 전문가에게 보여줄 수 있는 기능적 프로토타입을 신속하게 구축하여(예: UCSF 전문가에게 접근성 도구 보여주기) 아이디어를 검증하고 기존 솔루션을 식별하여 더 많은 시간을 투자하기 전에 파악합니다.

업무 스타일과 영향력

Claude.ai에서 계획, Claude Code에서 구축

이들은 먼저 Claude.ai로 브레인스토밍하고 계획을 세운 다음 구현을 위해 Claude Code로 이동하는 2단계 프로세스를 사용하며, 모든 것을 한 번에 출력하는 대신 속도를 늦추고 단계별로 작업하도록 요청합니다.

시각적 우선 접근 방식

이들은 종종 스크린샷을 사용하여 인터페이스가 어떻게 생겼으면 하는지를 Claude Code에 보여준 다음, 기능을 텍스트로 설명하는 대신 시각적 피드백을 기반으로 반복 작업을 진행합니다.

프로토타입 중심의 혁신

이들은 '바보 같은' 또는 '장난감' 프로토타입을 공유하는 것에 대한 두려움을 극복하는 것을 강조하는데, 이러한 데모를 통해 다른 사람들이 미처 생각하지 못했던 가능성을 볼 수 있기 때문입니다.

보안 및 규정 준수 인식

MCP 통합에 대한 우려

제품 변호사는 Claude Code를 사용하여 심층적인 MCP 통합의 보안 영향을 즉시 파악하고, AI 도구가 더 민감한 시스템에 액세스할 때 보수적인 보안 태세가 어떻게 장벽을 만들 수 있는지에 주목합니다.

규정 준수 도구의 우선순위

이들은 혁신과 위험 관리 사이의 균형을 인식하고 AI 기능이 확장됨에 따라 규정 준수 도구를 신속하게 구축할 것을 지지합니다.

법무 부서의 주요 팁

먼저 Claude.ai에서 광범위하게 계획하기

Claude의 대화형 인터페이스를 사용하여 전체 아이디어를 구체화한 다음 Claude Code로 이동하세요. 그런 다음 Claude에게 구현을 위한 단계별 프롬프트로 모든 것을 요약해 달라고 요청하세요.

점진적이고 시각적으로 작업하기

Claude Code에게 속도를 늦추고 한 번에 한 단계씩 구현해 달라고 요청하면 부담 없이 복사-붙여넣기를 할 수 있습니다. 스크린샷을 자유롭게 사용하여 원하는 인터페이스의 모습을 보여주세요.

불완전하더라도 프로토타입을 공유하세요.

'장난감' 프로젝트나 미완성된 작업을 숨기고 싶은 충동을 극복하세요. 프로토타입을 공유하면 다른 사람들이 가능성을 확인하고 일반적으로 상호 작용하지 않는 부서 간에 혁신을 촉발하는 데 도움이 됩니다.
