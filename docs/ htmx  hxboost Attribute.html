hx-boost 속성을 사용하면 일반 앵커와 양식 태그를 "부스트"하여 대신 AJAX를 사용할 수 있습니다. 이는 사용자가 자바스크립트를 활성화하지 않은 경우에도 사이트가 계속 작동한다는 좋은 대체 기능이 있습니다.

앵커 태그의 경우 앵커를 클릭하면 href 에 지정된 URL에 GET 요청이 실행되고 해당 URL이 푸시되어 기록 항목이 만들어집니다. 대상은 <body> 태그이며 innerHTML스왑 전략이 기본적으로 사용됩니다. click 트리거를 제외한 모든 태그는 적절한 속성을 사용하여 수정할 수 있습니다.

양식의 경우 요청은 method 속성의 메서드에 따라 GET 또는 POST 으로 변환되고 submit 에 의해 트리거됩니다. 이 경우에도 타겟은 페이지의 body 가 되며 innerHTML스왑이 사용됩니다. 그러나 URL은 푸시되지 않으며 히스토리 항목이 생성되지 않습니다. (URL을 푸시하려면hx-push-url 속성을 사용하면 됩니다.

다음은 몇 가지 부스트 링크의 예입니다:

<div hx-boost="true">
  <a href="/page1">Go To Page 1</a>
  <a href="/page2">Go To Page 2</a>
</div>

이러한 링크는 각 URL에 ajax GET 요청을 실행하여 본문의 내부 콘텐츠를 해당 URL로 대체합니다.

다음은 부스트된 양식의 예입니다:

<form hx-boost="true" action="/example" method="post">
    <input name="email" type="email" placeholder="Enter email...">
    <button>Submit</button>
</form>

이 양식은 지정된 URL에 ajax POST 요청을 발행하고 본문의 내부 콘텐츠를 해당 요청으로 대체합니다.

노트

  • hx-boost 는 상속되며 부모 요소에 배치할 수 있습니다.
  • 동일한 도메인으로 연결되고 로컬 앵커가 아닌 링크만 부스트됩니다.
  • 모든 요청은 AJAX를 통해 이루어지므로 리디렉션과 같은 작업을 수행할 때 이를 염두에 두어야 합니다.
  • 요청이 부스트된 앵커 또는 양식에서 발생하는지 확인하려면 요청 헤더에서 HX-Boosted 를 확인합니다.
  • 다음을 사용하여 하위 요소에서 부스트를 선택적으로 비활성화합니다. hx-boost="false"
