웹 컴포넌트를 처음 사용하시나요?

요약하자면, 드디어 자체 HTML 요소를만들어 원하는 프레임워크에서 사용할 수 있는 방법이 생겼습니다!

앵귤러, 뷰, 리액트와 같은 프레임워크의 인기 덕분에 컴포넌트 중심 개발은 우리 일상의 일부가 되었습니다. 컴포넌트는 스타일과 동작을 재사용 가능한 빌딩 블록으로 캡슐화하는 데 도움이 됩니다. 컴포넌트는 디자인, 개발, 테스트 측면에서 많은 의미가 있습니다.

하지만 안타깝게도 프레임워크별 컴포넌트는 여러 가지 면에서 단점이 있습니다:

  • 해당 컴포넌트가 설계된 프레임워크에서만 사용할 수 있습니다 🔒.
  • 수명이 해당 프레임워크의 수명으로 제한됨 ⏳
  • 새로운 프레임워크/버전이 나오면 컴포넌트를 업데이트하는 데 상당한 노력이 필요함 😭.

웹 컴포넌트는 이러한 문제를 해결합니다. 웹 컴포넌트는 모든 최신 브라우저에서 지원되며 프레임워크에 구애받지 않고 표준의 일부이므로 향후 수년 동안 지원될 것입니다.

이것이 바로 슈레이스의 기반 기술입니다.

어떤 문제를 해결하나요?

슈레이스는 프레임워크에 구애받지 않는 기술을 기반으로 전문적으로 디자인되고 고도로 사용자 정의 가능한 UI 컴포넌트 모음을 제공합니다. 왜 처음부터 디자인 시스템을 구축하는 데 수백 시간 이상을 소비해야 할까요? 왜 하나의 프레임워크에서만 작동하는 컴포넌트 라이브러리를 만들어야 할까요?

슈레이스를 사용하면 가능합니다:

  • 더 빠르게 빌드 시작(직접 버튼을 누를 필요 없음)
  • 동일한 UI 컴포넌트를 공유하는 서로 다른 프레임워크로 여러 앱을 빌드할 수 있습니다.
  • 기존 디자인에 맞게 컴포넌트를 완벽하게 커스터마이징할 수 있습니다.
  • 필요에 따라 컴포넌트를 점진적으로 채택(프레임워크를 버릴 필요 없음)
  • 기본 구성 요소를 다시 빌드하지 않고도 프레임워크 업그레이드 또는 전환 가능

조직에서 디자인 시스템을 구축하려는 경우, 슈레이스를 사용하면 수천 달러를 절약할 수 있습니다. 필요한 모든 기본 구성 요소가 바로 여기에 있으며, 브랜드에 맞게 맞춤 설정할 수 있습니다. 또한 웹 표준을 기반으로 구축되었기 때문에 브라우저는 앞으로도 계속 지원할 것입니다.

조직의 디자인 시스템을 위한 출발점으로 사용하든, 재미있는 개인 프로젝트를 위해 사용하든, 슈레이스로 할 수 있는 일에는 제한이 없습니다.

브라우저 지원

슈레이스는 다음 브라우저의 최신 두 버전에서 테스트되었습니다.

이전 버전의 중요한 버그 수정은 그 심각성과 영향에 따라 해결됩니다.

IE11 또는 크롬 엣지 이전 버전을 지원해야 하는 경우 이 라이브러리는 적합하지 않습니다. 레거시 브라우저에서 웹 컴포넌트를 어느 정도 폴리필링할 수는 있지만, 이를 지원하는 것은 이 프로젝트의 범위를 벗어납니다. 그런 브라우저에서 슈레이스를 사용한다면 곤란한 시간을 보내게 될 것입니다. ⛷

저작자 표시

슈레이스가 가능하도록 도와주신 다음 프로젝트와 개인에게 특별히 감사드립니다.

  • 컴포넌트는 Lit로빌드됩니다.
  • 컴포넌트 메타데이터는 사용자 정의 요소 매니페스트 분석기에의해 생성됩니다.
  • 문서는 11ty에서제공
  • CDN 서비스는 jsDelivr에서제공합니다.
  • 색상 기본 요소는 Tailwind에서영감을 얻었습니다.
  • 아이콘은 부트스트랩 아이콘제공
  • 홈페이지 일러스트레이션은 unDraw제공
  • 드롭다운, 툴팁 등의 위치 지정은 플로팅 UI에서처리합니다.
  • 애니메이션은 animate.css제공
  • 검색은 Lunr제공
  • 신발끈 로고는 신발끈 하나로 디자인한 Adam K Olson의작품입니다.
