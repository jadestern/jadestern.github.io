안녕하세요, React 개발자 여러분! 이제 오래된 습관을 버리고 미래로 나아갈 때입니다.

우리는 2023년에 살고 있으며, React에서 조건부 렌더링을 처리하는 데 "&&" 연산자를 사용하는 것보다 더 좋은 방법이 있습니다.

이 글에서는 코드를 더 깔끔하고 읽기 쉽게 만드는 조건부 렌더링에 대한 더 우아하고 표현력 있는 접근법을 보여드리겠습니다.

이제 "&&"에 작별을 고하고 React에서 보다 현대적인 조건부 렌더링 방식을 도입해 봅시다.

시작해 봅시다!

예전 방식: "&&"를 사용한 조건부 렌더링:

과거에는 조건에 따라 컴포넌트나 요소를 조건부로 렌더링하기 위해 "&&" 연산자를 많이 사용했습니다.

예를 들어 다음과 같은 코드를 작성했습니다:

function MyComponent({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn && <p>Welcome back, User!</p>}
    </div>
  );
}

이 접근 방식은 작동하지만 여러 조건이나 더 복잡한 렌더링 로직을 처리할 때는 번거롭고 가독성이 떨어질 수 있습니다.

다행히도 React는 발전해 왔으며 이제 더 우아한 솔루션이 있습니다.

현대적인 방법: 삼항 연산자 또는 논리적 OR을 사용한 조건부 렌더링:

"&&" 연산자에만 의존하는 대신, 조건부 렌더링을 위해 삼항 연산자 또는 논리 OR(||) 연산자의 힘을 활용할 수 있습니다.

두 가지 옵션을 모두 살펴보겠습니다.

  1. 삼항 연산자를 사용한 조건부 렌더링:

삼항 연산자를 사용하면 간결한 조건식을 작성할 수 있으므로 코드의 표현력이 향상됩니다. 다음은 예제입니다:

function MyComponent({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <p>Welcome back, User!</p> : null}
    </div>
  );
}

이 코드에서 "isLoggedIn" 조건이 참이면 환영 메시지와 함께 <p> 요소를 렌더링합니다. 그렇지 않으면 null을 렌더링하여 요소를 효과적으로 숨깁니다.

2. 논리 OR(||) 연산자를 사용한 조건부 렌더링:

논리 OR(||) 연산자는 조건부 렌더링에 대한 또 다른 접근 방식을 제공합니다.

조건이 거짓일 때 기본값을 지정할 수 있습니다. 다음은 예시입니다:

function MyComponent({ username }) {
  return (
    <div>
      <p>Welcome, {username || "Guest"}!</p>
    </div>
  );
}

이 코드에서 "username" 프로퍼티가 제공되면 <p> 요소에 렌더링됩니다. "username" 프로퍼티가 거짓인 경우(예: 빈 문자열 또는 정의되지 않음) 논리적 OR 연산자는 기본값인 "Guest"를 반환합니다.

최신 접근 방식의 장점:

삼항 연산자 또는 논리 OR 연산자를 사용하면 기존 "&&" 접근 방식에 비해 몇 가지 이점이 있습니다:

  1. 가독성: 특히 더 복잡한 조건이나 여러 렌더링 옵션을 다룰 때 코드가 더 표현력이 풍부해지고 이해하기 쉬워집니다.
  2. 유연성: 삼항 연산자를 사용하면 참과 거짓 조건을 모두 처리할 수 있으므로 렌더링 로직을 더 세밀하게 제어할 수 있습니다.
  3. 기본값: 논리 OR 연산자는 조건이 거짓일 때 기본값을 지정할 수 있는 편리한 방법을 제공하여 추가 로직의 필요성을 줄여줍니다.

2023년에는 React에서 조건부 렌더링을 위한 기존의 "&&" 접근 방식에 작별을 고할 때입니다.
